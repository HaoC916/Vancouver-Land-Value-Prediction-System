# Contributing

## Branch Strategy
- `main` — Stable, release-ready snapshots
- `dev` — Integration branch for ongoing work
- `feature/<short-name>` — Feature branches for individual tasks

## Pull Request Checklist
- No large data files or artifacts included
- README updated if behavior or structure changes
- Formatting/linting checked as applicable

## Commit Message Conventions
Use a short prefix and a concise summary, for example:
- `chore: ...`
- `data: ...`
- `model: ...`
- `eval: ...`
- `docs: ...`
